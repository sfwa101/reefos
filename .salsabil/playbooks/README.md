# Playbooks

Operational procedures. Each playbook is a numbered, step-by-step runbook with preconditions, steps, rollback, and verification.

## Index

- `01-add-capability.md` — declare and roll out a new capability.
- `02-add-event.md` — register a new event in the catalog.
- `03-add-domain.md` — bootstrap a new sovereign domain.
- `04-schema-migration.md` — author a safe schema migration.
- `05-incident-response.md` — handle a production incident.
- `06-deprecate-contract.md` — retire a published contract safely.

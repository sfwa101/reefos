# Domain Memories

Each domain owns a memory file mirroring `_template.md`. Files are created the first time a domain undergoes material change under sovereign governance.

See `architecture/02-ownership-map.md` for the canonical list and paths.

## Conventions

- File name: `<domain>.md` (lowercase, kebab-case).
- Authoritative for that domain. Conflicts with code resolve by correcting the code.
- Updated in the same PR as code changes affecting the domain.

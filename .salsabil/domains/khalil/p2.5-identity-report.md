## Database (migration `20260516_p2.5_identity`)
- **`khalil_identity_state`** ‚Äî projection. Columns: `current_level`,
  `current_score`, `window_30d`, `window_90d`, `window_180d`, `observed_days`,
  `last_computed_at`, `updated_at`. RLS: owner SELECT.
- **`khalil_identity_event`** ‚Äî append-only log. Immutability triggers
  (`khalil_identity_event_no_update` / `_no_delete`) block UPDATE/DELETE.
  Indexed by `(user_id, created_at DESC)`. RLS: owner SELECT.
- **`khalil_adherence_daily`** extended with `identity_score`,
  `identity_level_snapshot` (for analytics replay).
- **`khalil_recompute_identity(p_user_id)`** ‚Äî SECURITY DEFINER. Computes
  rolling windows, blends with weights (0.5/0.3/0.2), applies recovery
  softening, resolves level against thresholds, anti-leap (‚â§1 level up),
  hard-recovery cannot demote alone, upserts projection, stamps adherence
  day, inserts identity event on transition only.
- **Subscribers**: triggers on `khalil_adherence_daily` (INSERT/UPDATE)
  and `khalil_recovery_state` (INSERT/UPDATE) ‚Üí recompute.
- Helpers `level_idx(text)` / `level_name(int)` ‚Äî IMMUTABLE SQL.

  append-only event log.

# Phase P0.1 — Migration Impact Report

## User-visible impact

| Surface                          | Before                          | After                                  | Risk |
|----------------------------------|---------------------------------|----------------------------------------|------|
| Route `/_app/maeen` (`/maeen`)   | Renders `MaeenHub` from khalil  | Renders identical `MaeenHub` from maeen | None — byte-equivalent component. |
| Route `/khalil`                  | 404 (until previous patch) → governance landing | Governance landing (unchanged) | None. |
| OS Switcher tile "خليل"          | Opens `/khalil` → governance landing, labeled "السوبر-أب الموحد" (wrong) | Same route, correct label "نظام التحول السيادي" | Cosmetic correction only. |
| OS Switcher tile "معين"          | Did NOT exist as a top-level tile (only via `OS_MODULES`) | New first-class tile → `/maeen` | Additive; default-launcher pins are unaffected. |
| Reef storefront `LayoutFactory`  | Renders `MaeenLauncherGrid`     | Renders the same component from new path | None — component logic untouched. |
| SDUI slug `khalil_hub`           | Used by `MaeenLauncherGrid`     | Still used (DB row unchanged)         | Tracked — see coupling-risk §1. |

## Code impact

- **Imports rewritten:** 2 (`_app.maeen.tsx`, `LayoutFactory.tsx`).
- **Files deleted:** 2 (`src/apps/khalil/pages/Hub.tsx`, `src/apps/khalil/components/MaeenLauncherGrid.tsx`).
- **Files added:** 5 (3 under `src/apps/maeen/`, plus 2 reports under `.salsabil/`).
- **Files re-written in place:** 3
  (`src/apps/khalil/README.md`, `src/core/identity/osCompanies.ts`,
  `src/components/AppSwitcherSheet.tsx` — `APP_ROUTES` only).
- **Kernel changes:** none. `osCompanies.ts` is a pure data module
  under `src/core/identity/` and is treated as identity registry, not
  kernel mechanism. No registries, resolvers, or renderers altered.

## Database impact

**None.** The SDUI row `sdui_layouts.slug = 'khalil_hub'` is untouched.
Migrating it is deferred to ADR-0004 to keep this phase code-only.

## Persisted-client-state impact

- `localStorage` key `salsabil_active_os_company` — values still valid
  (`khalil` remains a known id; meaning has changed but storage shape
  has not).
- `localStorage` key `salsabil.defaultLauncher` — any pin made against
  the *old* "khalil" tile (intending Maeen) will now open the Khalil
  landing. **Mitigation:** the switcher now shows a distinct "معين"
  tile the user can re-pin. No data migration is required or possible
  from the server side (client-local storage).

## Build / type-check impact

- `bunx tsc --noEmit` must pass (acceptance gate in ADR-0003).
- One forced compile-time check: the new `maeen` member of
  `OSCompanyId` forces `APP_ROUTES` exhaustiveness, surfacing any
  future divergence at build time rather than at runtime.

## Reversibility

Pure code move. Reversible by `git revert` on the P0.1 commit.
No schema migration, no destructive data action.

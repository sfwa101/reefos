## Scope

Sovereign Habit pillar + extended adherence projection + identity-engine
scaffolding. Reuses the gateway-monopoly + append-only law established
in P2.2 (Prayer).

## Database (migration)

- `khalil_habit_definition` — slug-unique per user, archive-only update,
  hard-delete forbidden by trigger.
- `khalil_habit_completion` — append-only, dedup on (user, habit, date)
  + `client_event_id`. RLS owner-only. Immutability trigger.
- `khalil_adherence_daily` extended with `prayer_score`, `habit_score`,
  `combined_score`, `habit_completed_count`, `habit_target_count`.
- `khalil_recompute_adherence(p_user_id, p_date)` rewritten to fold both
  pillars and remain idempotent (full recompute per day).
- AFTER-INSERT trigger on `khalil_habit_completion` re-runs the recompute.

## Capabilities

Registered in `KHALIL_CAP` (server-side gate via
`requireKhalilCapability(cap)` middleware; unknown keys fail closed):

- `khalil.habit.define.write`
- `khalil.habit.archive.write`
- `khalil.habit.complete.write`
- `khalil.habit.read`
- `khalil.analytics.private.read` (replay job)

## Events

Names added to `KHALIL_EVENT`. Payload contracts in
`src/core/khalil/habit/events.ts`:
- `khalil.habit.defined`
- `khalil.habit.archived`
- `khalil.habit.completed`

Subscriber contract `handleHabitCompleted` is idempotent on event id and
calls the identity-engine stub (`noopIdentityHook`) — no level mechanics.

## Runtime (pure TS)

- `validateDefineHabit` — slug/cadence/target verdicts.
- `validateCompleteHabit` — partial bounds, same-day + recovery rules.
- `classifyHabitInsertError` — 23505 → replay/semantic/fresh.
- `computeHabitDayView` — deterministic view composer.
- `computePrayerAdherence`, `computeHabitAdherence`,
  `computeCombinedAdherence` — pure deterministic math, mirrored in SQL.

## Gateways (server fns)

- `defineHabitFn`, `archiveHabitFn`
- `completeHabitFn` (ownership backstop + dedupe)
- `readHabitsTodayFn`
- `replayHabitProjectionFn`

All carry `requireSupabaseAuth` + `requireKhalilCapability(...)`. UI
imports only via `@/core/khalil`.

## UI

- `khalil.habit.today` block: list, quick-complete, optimistic update,
  pending-sync chip, loading/empty/error via primitives.
- `composeKhalilHome` mounts welcome → prayer → habit (static array; no
  reorder logic per contract).
- `/khalil/habits` renders the block as its own page.

## Offline

`khalilOfflineQueue` was already IDB-backed in P2.2 (`idb-keyval`). Habit
completions reuse the same singleton with `KHALIL_CAP.HABIT_COMPLETE_WRITE`
+ `idempotencyKey = clientEventId`. FIFO replay survives refresh.

## Identity-engine scaffolding

`src/core/khalil/identity/index.ts` pins the level ladder
(`seed → sprout → branch → tree → forest`), signal shape, and a
deterministic `previewIdentityLevel` stub returning `"seed"`. No
transitions wired. Real engine deferred behind ADR-K001.

## Tests

- `validateCompleteHabit.test.ts` — partial bounds, future/stale/recovery.
- `dedupeHabitCompletion.test.ts` — 23505 classifier.

logs; calling N times with the same args yields identical rows
(verified by construction + projection columns equal `EXCLUDED.*`).

## Domain boundary rules now in force

| From → To                          | Allowed? | Note                                  |
|------------------------------------|----------|---------------------------------------|
| `src/apps/maeen/**`  → `@/core/*`  | ✅       | Kernel + capability consumption.      |
| `src/apps/maeen/**`  → `@/apps/khalil/**` | ❌ | Civilization-to-civilization leakage. |
| `src/apps/khalil/**` → `@/core/khalil/*`  | ✅ | Sovereign gateway (when implemented). |
| `src/apps/khalil/**` → `@/apps/maeen/**`  | ❌ | Same rule, reverse direction.         |
| Kernel (`src/core/**`) → `src/apps/**`    | ❌ | Art. IX — Kernel Minimalism.          |
| `LayoutFactory` → `@/apps/<civ>/components/*` | ⚠️ | Allowed but transitional; long-term these become SDUI blocks registered via the runtime-ui registry, not direct imports. Tracked as a Maeen-domain follow-up. |

## 2. Replay topology

Declared in `src/core/khalil/replay/topology.ts`. Every projection rebuilds
from append-only event tables in deterministic order:

```
adherence       ← khalil_habit_completion        → khalil_adherence_daily
streaks         ← khalil_habit_completion        → khalil_streak_state
identity        ← khalil_adherence_daily         → khalil_identity_state
recovery        ← khalil_recovery_event          → khalil_recovery_state
prayer          ← khalil_prayer_log              → khalil_prayer_daily
workoutVolume   ← khalil_workout_set             → khalil_workout_volume_daily
weightTrend    ← khalil_weight_measurement      → khalil_weight_projection
```

`runReplay({ dispatch, onProgress })` walks the topology in declared order,
emits per-projection progress (key/index/total/durationMs/ok/error), and
honors `stopOnError`. `dispatch` is injected so this module stays free of
direct Supabase usage and cross-domain wiring — callers map each key to its
gateway server fn.

Drift detection (`detectDrift(key, live, replayed)`) hashes both snapshots
with a stable key-sorted FNV-1a checksum and reports row-level mismatch
count. Drift correction policy remains **replay, not patch** (Art. IV,
anti-pattern 07).

## 3. Offline durability contract

| Guarantee | Mechanism |
|---|---|
| Survives refresh/restart | `idb-keyval` persistence (P2.2) |
| Schema versioning | `KHALIL_QUEUE_SCHEMA_VERSION = 1` exported and stamped in diagnostics |
| Corruption recovery | Pre-existing try/catch on hydrate → start clean |
| Max queue size | `enforceMaxSize` drops oldest first (FIFO preserved), default 500 |
| Orphan cleanup | `pruneOrphans` drops intents older than 14d |
| Retry backoff | `backoffMs(attempt)` = min(300s, 1000·2^attempt) + jitter |
| FIFO preservation | Eviction only at the head; new intents always pushed to tail |
| Conflict rules | Server timestamp wins; duplicate intents are server-side no-ops via `idempotencyKey` |
| Diagnostics | `diagnose(buf)` → `{schemaVersion, size, oldestAgeMs, capabilities}` |

Replay batching, background-sync, and version migrations live in
`khalilOfflineQueue` (P2.2 contract) and consume these helpers without
changing its exported shape.

## 4. Governance enforcement

`scripts/khalil-governance.mjs` runs over the entire repo and exits non-zero
on any violation. Rules:

1. No `@/integrations/supabase/client` import inside `src/apps/khalil/**`
2. No cross-domain imports between sibling domains under `src/core/khalil/<X>/`
   (shared modules: `capabilities`, `events`, `i18n`, `middleware`, `gateway`,
   `replay`, `offline`, `analytics`, `orchestrator`, `queryKeys`)
3. No hardcoded Arabic literals in Khalil TSX (must route through `kt()`)
4. No `*.server.ts` imports from `src/apps/khalil/**`
5. No chart library imports in `src/apps/khalil/blocks/**` (except
   `*Chart*` / `Insights*` files which are lazy-loaded off the home route)
6. No `coach/runtime/prompts.server` import outside the coach gateway

Inline opt-out: `// khalil-governance-allow: rule-2` (only for ADR-tracked
debt — see ADR index below). Eight legacy cross-domain imports of shared
schema primitives (`isoDateSchema`, `clientEventIdSchema`, type-only
`RecoveryMode`, `KhalilIdentityLevel`) are tagged under
**ADR-2026-P2.8-01**.

Current run: **clean** (0 active violations, 8 ADR-tracked opt-outs).

### Recommended CI wiring

## 7. PWA / SW posture

The project intentionally does **not** ship a service worker (see Lovable
PWA guidance — service workers in iframes cause stale-cache issues during
preview). Offline behaviour for Khalil is achieved entirely through:

- IndexedDB-backed write queue (`khalilOfflineQueue`)
- Stale-while-revalidate via React Query cache
- Server timestamp wins on reconnect

If Beta later requires a SW, follow the kill-switch + iframe-guard pattern
from the PWA knowledge file. Not in scope for P2.8.

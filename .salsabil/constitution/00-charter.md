# Constitutional Charter

> The Constitution is sovereign. Features serve it. AI obeys it. Code conforms to it.

## Articles

1. **Controlled Integrated Evolution** — the system evolves only through reviewed, reversible, audited steps. No silent drift.
2. **Evolutionary Sovereign Architecture** — every layer is independently replaceable behind a stable contract.
3. **Capability-First Authorization** — all access is gated by named capabilities, never by roles, strings, or flags.
4. **Event-Driven Truth** — state is a projection of an append-only event log.
5. **Domain Sovereignty** — domains own their data, contracts, and lifecycle; cross-domain access is via published contracts only.
6. **Stable Contracts, Replaceable Implementations** — contracts version, implementations swap.
7. **Anti-Hardcoding** — no tenant, role, section, locale, price, or copy hardcoded in kernel or shared code.
8. **Dashboard Composition** — all admin UI is descriptor-driven (SDUI); no bespoke admin pages.
9. **Kernel Minimalism** — the kernel is mechanism; policy lives outside.
10. **Governance-First Engineering** — governance documents precede code; code matches governance, not the inverse.
11. **AI Execution Constraints** — AI is an advisor; mutations require human or capability-bearing operator approval.
12. **Runtime Memory Persistence** — every notable runtime decision is observable, traceable, and reconstructable.

## Precedence

`constitution/` > `architecture/` > `runtime/` > `domains/` > implementation code.

A conflict resolves upward. Implementation never overrides higher layers — it is corrected.

## Amendment

Amend only via ADR tagged `amends-constitution`, with rationale, blast radius, and migration plan.

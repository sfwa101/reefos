# ADR-0003 — Phase P0.1: Maeen ↔ Khalil decoupling

| Field | Value |
|---|---|
| **Status** | `Accepted` |
| **Date** | 2026-05-16 |
| **Phase** | Khalil P0.1 — Boundary Correction |
| **Supersedes** | none |
| **Refines** | ADR-0002 (Khalil sovereign domain) |

## Context

ADR-0002 ratified Khalil as the sovereign **transformation OS** and
flagged that the legacy `src/apps/khalil/` directory was hosting
**Maeen** content (the super-app launcher hub). That coupling caused:

1. **Domain ambiguity** — two different civilizations (transformation OS
   and unified launcher) sharing one directory and one `OS_COMPANIES`
   entry tagged "السوبر-أب الموحد".
2. **Cross-domain leakage** — `LayoutFactory` (Reef storefront) imported
   `MaeenLauncherGrid` from `@/apps/khalil/...`, encoding the wrong
   ownership into the kernel-adjacent registry.
3. **Capability dilution** — `OS_MODULES.shared_by` listed `khalil` as a
   consumer of `catalog` and `wallet` because the entry actually meant
   Maeen. This poisoned future capability resolution.

P0.1 corrects boundaries **without** adding features, UI, or business
logic. It is a pure architectural cleanup pass.

## Decision

1. **Move Maeen content** out of `src/apps/khalil/` into a new sovereign
   directory `src/apps/maeen/`:
   - `pages/Hub.tsx` → `src/apps/maeen/pages/Hub.tsx`
   - `components/MaeenLauncherGrid.tsx` → `src/apps/maeen/components/MaeenLauncherGrid.tsx`
   - Add `src/apps/maeen/README.md` declaring the boundary.
2. **Re-point importers** that referenced the old paths:
   - `src/routes/_app.maeen.tsx`
   - `src/apps/reef-al-madina/.../LayoutFactory.tsx`
3. **Delete** `src/apps/khalil/pages/` and `src/apps/khalil/components/`.
   The Khalil app dir now holds only `README.md` (presentation scaffold).
4. **Re-label `OS_COMPANIES`**:
   - `maeen` added as a first-class civilization (status `live`, icon
     `Layers`, tagline "بوابة الإمبراطورية الموحّدة").
   - `khalil` re-tagged with its real identity ("نظام التحول السيادي
     الموجَّه بالذكاء", icon `Compass`, status `design`).
5. **Re-balance `OS_MODULES.shared_by`** to reflect real consumers:
   - `wallet`, `catalog` → `khalil` removed, `maeen` added where it
     actually consumes.
   - `hakim_ai` → `khalil` added (the transformation OS legitimately
     consumes Hakim for the AI coach).
6. **Add the `/maeen` route** to `APP_ROUTES` in `AppSwitcherSheet` so
   the switcher resolves Maeen to its own surface.
7. **Defer the DB-side rename** of the SDUI slug `khalil_hub` →
   `maeen_hub`. That is a schema migration and requires its own ADR
   (see Follow-ups). The code retains the legacy slug constant with a
   pointer to `p0.1-coupling-risk.md`.

## What is explicitly NOT in scope

- No new Khalil features, screens, or runtime code.
- No new Maeen features.
- No kernel changes (`src/core/runtime-ui/`, `src/core/capabilities/`,
  etc. are untouched beyond the `osCompanies.ts` data module).
- No DB migrations.
- No UI redesign or visual changes.

## Alternatives considered

| Alternative | Rejected because |
|---|---|
| Leave Maeen inside `src/apps/khalil/` and rename later | Perpetuates ambiguity; every new contributor would re-read the wrong identity from the directory name. |
| Migrate the `sdui_layouts` row `khalil_hub` → `maeen_hub` in this ADR | Mixes a code-only cleanup with a schema migration; violates Art. X order of operations. Tracked as follow-up. |
| Make `khalil` and `maeen` share an internal launcher registry | Couples two sovereign civilizations through a private surface; violates Art. V (Domain Sovereignty). |

## Consequences

- ✅ Khalil and Maeen are visibly distinct in code, governance docs, and
  the OS Switcher.
- ✅ The kernel-adjacent `LayoutFactory` no longer reaches into a
  mislabeled domain.
- ✅ `OS_MODULES` now accurately models cross-domain capability sharing.
- ⚠️ The SDUI row slug `khalil_hub` still exists in the DB. Documented
  in `p0.1-coupling-risk.md`; will be addressed by ADR-0004.
- ⚠️ Existing `localStorage` `salsabil.defaultLauncher` entries that
  pinned the *old* "khalil" tile (which actually opened Maeen content)
  will now route the user to the new Khalil landing. Acceptable: the
  user can re-pin Maeen from the switcher.

## Acceptance criteria

- [x] `src/apps/khalil/pages/` and `src/apps/khalil/components/` deleted.
- [x] `src/apps/maeen/{pages,components,README.md}` exist and contain

- [x] No remaining import from `@/apps/khalil/components/...` or
      `@/apps/khalil/pages/...` anywhere in `src/`.
- [x] `OS_COMPANIES` contains both `maeen` and a correctly-tagged
      `khalil`.
- [x] `APP_ROUTES` in `AppSwitcherSheet.tsx` covers every `OSCompanyId`.

- [x] Reports produced under `.salsabil/domains/khalil/`:
      `p0.1-boundary-correction.md`, `p0.1-migration-impact.md`,
      `p0.1-coupling-risk.md`.
- [x] `.salsabil/architecture/02-ownership-map.md` updated with both
      civilizations.

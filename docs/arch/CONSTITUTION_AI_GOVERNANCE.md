# AI GOVERNANCE & ARCHITECTURAL INTEGRITY PROTOCOL — SALSABIL OS

1. **AI Sovereignty:** No AI Agent is permitted to make autonomous architectural decisions or create new patterns outside the established Constitution.

2. **Mandatory Audit:** Before writing any code, a read-only scan of the impacted Runtime boundaries and Capabilities is required.

3. **Runtime-First Policy:** Features MUST be built in this order: Core Runtime -> Capability -> Schema -> UI. Never UI-first.

4. **Sovereign Isolation:** The UI is strictly forbidden from importing the Supabase client or performing direct DB/Auth mutations. All data flow MUST transit through Sovereign Gateways.

5. **No Framework Worship:** If a framework feature (React/Supabase) conflicts with Constitutional isolation, the Constitution prevails.
